# 安全模型

| 配置安全 | ✅ 原子写入 | 配置先写入临时文件，再原子重命名 |

## 已知局限

### 数据库加密

SQLite 数据库（`x-ui.db`）**未加密存储**。TG Bot 令牌使用 AES-256-GCM 单独加密存储（密钥独立文件），但其他设置和入站配置以明文存储。

**缓解措施**：限制数据库文件的文件系统访问权限。在生产环境中，确保只有 x-ui 进程用户可读取。

### Xray gRPC API

Xray 内部统计 API 暴露在 `127.0.0.1`，无需身份验证。这是 Xray 的标准安全模型 — 仅本地进程可连接。

### 会话密钥

会话签名密钥在首次运行时自动生成并存储在数据库中。如果攻击者获取了数据库读取权限，即可伪造会话 Cookie。可通过删除 "secret" 设置行并重启面板来重置。

### HTML 模板中的 CSRF Token

CSRF Token 通过模板变量注入到 HTML 页面中，这是前端在请求中包含它的必要方式。所有状态变更操作（POST/PUT/PATCH/DELETE）都会验证该 Token。

## 安全建议

1. **为管理面板启用 HTTPS**（使用内置 ACME 客户端或反向代理）
2. **使用防火墙限制管理端口访问**（例如仅允许特定 IP）
3. **使用强密码**（面板强制执行 — 最少 8 位，需包含 4 类字符中的至少 3 类）
4. **设置自定义 WebBasePath**以隐藏面板登录页面
5. **定期更新 Xray-core**（使用内置更新机制，SHA-256 校验下载）
6. **监控安全日志**以及时发现未授权访问尝试
